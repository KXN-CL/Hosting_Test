About Project

프로젝트 소개

이 프로젝트는 로컬 환경에서 개발 보조 AI와 워크플로우 자동화를 결합해, 아이디어를 더 빠르게 구현하기 위한 개인용 개발 시스템을 설계하는 작업이다.

로컬 중심 구조

클라우드 의존도를 줄이고, 보유한 GPU 자원을 활용해 개인 전용 AI 개발 환경을 구축하는 방향을 채택했다.

세션 단위 작업

단순 질의응답이 아니라, 계획 → 생성 → 검증 → 수정의 흐름을 하나의 세션 단위로 관리하는 구조를 목표로 한다.

포트폴리오이자 실험

국비 수업의 포트폴리오 성격을 가지면서도, 실제로 확장 가능한 개인 개발환경의 설계 실험을 겸한다.

System Architecture

시스템 설계 방향

초기 버전은 복잡한 병렬 구조보다, 이해와 유지보수가 쉬운 선형 워크플로우를 중심으로 설계한다. 이후 필요 시 검증 계층과 자동화를 확장하는 방향을 고려한다.

결과물 정리

세션 매니저 중심

하나의 긴 세션 안에서 작업을 모두 처리하기보다, 세션 단위로 맥락을 관리하며 필요한 문서와 산출물을 재활용하는 구조를 지향한다.

검증 결과의 재순환

생성된 결과물은 바로 종료하지 않고, 검증 계층을 거쳐 오류나 미완성이 발견되면 수정 루프로 다시 연결되도록 설계한다.

Technical Direction

기술 방향

현재는 실제 하드웨어 제약 안에서 구동 가능한 모델, 컨텍스트 길이, 검증 구조를 함께 고려해 방향을 잡고 있다.

Ubuntu 24.04
RTX 3090 중심 로컬 추론
Qwen 3.5 27B 4bit 계열
64K 토큰 단위 세션 설계
검증 자동화 구조
하이브리드 확장 가능성

운영 환경

Ubuntu 24.04를 중심으로 로컬 실행 환경을 구성하고, 실사용 가능한 자동화 기반을 다지는 방향이다.

추론 자원

RTX 3090을 중심으로 로컬 모델 추론을 우선 고려하며, 필요 시 보조 자원과 역할 분리를 검토한다.

모델 및 세션 설계

Qwen 3.5 27B 4bit 계열과 64K 수준의 세션 단위를 기준으로, 현실적인 작업 범위를 설계하고 있다.

MVP Principles

MVP 설계 원칙

초기 단계에서는 모든 기능을 한 번에 구현하지 않고, 가장 작은 단위의 검증 가능한 구조부터 구축하는 전략을 택한다.

최소 검증 루프 우선

생성 후 검증, 실패 시 수정이라는 핵심 흐름부터 구현한다.

지원 범위 최소화

언어와 외부 도구를 제한해 유지보수 부담을 낮춘다.

복잡한 UI 검증 후순위

이미지·스크린샷 기반 프론트엔드 검증은 MVP에서 제외한다.

텍스트/코드 중심 시작

처음에는 텍스트와 코드 산출물 검증에 집중한다.

선형 워크플로우 채택

복잡한 다중 워커보다 단순하고 명확한 구조부터 만든다.

점진적 확장

플러그인형 확장과 계층적 검증 구조는 이후 단계에서 추가한다.

Roadmap

향후 계획

현재 설계 산출물을 바탕으로, 실제 구동 가능한 최소 버전을 구현한 뒤 점진적으로 자동화 수준을 높일 예정이다.

01

세션 매니저 구체화

세션 단위 컨텍스트 관리와 작업 흐름 분기를 명확히 정의한다.

02

아티팩트 Writer / Loader 설계

문서와 결과물을 저장·재사용할 수 있는 구조를 구상한다.

03

검증 루프 MVP 구현

최소 기능의 생성-검증-수정 사이클을 실제 코드로 구현한다.

04

Git / GitHub 연동

버전 관리 및 포트폴리오 관리 흐름을 프로젝트 운영과 연결한다.

05

하이브리드 실험

복잡한 작업은 외부 API와 연계하는 구조도 실험 대상으로 검토한다.

06

실제 워크플로우 적용

학습용을 넘어 실제 개인 개발 루틴에 투입 가능한 수준을 목표로 한다.
